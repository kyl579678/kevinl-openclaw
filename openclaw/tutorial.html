1. 電腦配備選擇：三種方案

方案一：買一台全新專用電腦 ⭐ 推薦

買一台便宜的迷你電腦，專門拿來跑 OpenClaw，放在角落 24 小時運作。

推薦規格：

項目 最低 建議
CPU Intel N100 / i3-1305U Intel i3 以上
RAM 8GB 16GB
SSD 128GB 256GB+
GPU 不需要 不需要
系統 Windows 11 Windows 11

⚠️ 重要： Windows 11 新機預設會開啟 S Mode，須在 Microsoft Store 免費切換為完整版才能安裝 WSL2。建議直接選已預裝完整 Windows 11 的機型。

實際範例： - Acer Revo Box RB102（Intel i3-1305U / 8GB RAM / 128GB SSD / Windows 11）— 約 NT$10,900 - 升級至 16GB RAM 後總花費約 NT$12,000-13,000 - 放在桌角 24 小時運作，耗電約 10-15W，每月電費 NT$30-50

優點： 簡單、乾淨、穩定，資安風險最低（完全隔離），自由度最高（整台機器都是你的）

缺點： 需要一次性的硬體投資（約 NT$12,000-15,000）

方案二：用現有電腦 + VM / Docker 隔離

不改變現有電腦，在上面安裝 VM（虛擬機）或 Docker 來跑 OpenClaw，與日常使用環境隔離。

VM 選項：

  • Windows 內建 Hyper-V（Pro/Enterprise 版）
  • 或 VirtualBox（免費，適用所有 Windows 版本）

Docker 選項： - Docker Desktop for Windows - 適合有技術背景的使用者

優點： 不用買新硬體

缺點：

  • 安裝較複雜（需學習 VM 或 Docker 操作）
  • VM 佔用系統資源，日常電腦效能受影響
  • 資安風險比方案一高（仍與主系統共享硬體）
  • 某些情況下 WSL2 / Docker 網路設定容易出問題

方案三：雲端租用伺服器（VPS / 雲端主機）

把 OpenClaw 裝在租用的雲端伺服器上，24 小時運行，不需要自己的硬體。

選項：

Oracle Cloud Always Free 免費，ARM 架構，穩定性視情況 NT$0/月
Hetzner Cloud 性價比高，€3-4/月起 NT$150-300/月
AWS EC2 / Lightsail 生態完整，彈性大 NT$300-1000/月
GCP Compute Engine Google 生態整合 NT$300-1000/月

優點： 初期零硬體成本，可隨時升降配

缺點：

  • 需要學習 SSH、VPS 管理、網路安全設定
  • 資料在第三方伺服器（資安風險最高）
  • 月費長期累積，1-2 年後可能比買硬體還貴
  • 需要另架 Tailscale 或 Cloudflare Tunnel 讓你能從外面連回家中裝置

2. 模型選擇：Claude API vs MiniMax API

在設定 OpenClaw 之前，你需要先選擇一個 AI 模型背後的 API 服務商。以下是這兩大方案的詳細比較。

2.1 Claude API（Anthropic）

申請網站： console.anthropic.com

Claude 是目前市面上推理能力最強的模型之一，特別適合複雜的分析、程式寫作、和長上下文任務。

主要模型：

模型 特色 速度 價格
Claude Sonnet 4.5 速度與能力平衡，適合日常使用 ⭐⭐⭐⭐⭐ 低
Claude Opus 4.5 最強推理能力，適合複雜任務 ⭐⭐ 高
Claude Haiku 3.5 最便宜，適合簡單任務 ⭐⭐⭐⭐⭐ 極低

優點：

  • 模型能力頂尖，長上下文表現優秀（支援 200K token）
  • 工具呼叫（Tool Use）成熟穩定
  • 有詳細的 API 文件和開發者社群
  • 可設定消費上限，避免意外帳單

缺點：

  • 費用較高（實際使用約 USD $10-20/天）
  • 需要信用卡才能啟用 API
  • 回應速度在台灣/中國可能受網路延遲影響

2.2 MiniMax API（國際版）

申請網站： platform.minimax.io/subscribe/token-plan

MiniMax 是中國領先的 AI 公司，專注多模態模型（文本、語音、影像、音樂）。在 OpenClaw 生態中被廣泛使用，性價比極高。

申請流程：

  1. 前往 platform.minimax.io/subscribe/token-plan 選購 TokenPlan
  2. TokenPlan 是一種固定費用訂閱方案，不同方案有不同的用量上限和速度限制（詳見下表）
  3. 訂閱後進入 Dashboard → 「API Keys」→ 建立並複製 API Key
  4. 模型名稱使用 MiniMax-M2.5（或 MiniMax-M2.7 最新旗艦版）

⚠️ TokenPlan 方案價格請至官網確認，以下為參考範圍（建議選擇 $100-800/年的方案，享有較高用量上限和優先速度）：

維度 基本方案 進階方案 專業方案
年費（參考） ~$100-300/年 ~$300-500/年 ~$500-800/年
用量上限 較低 中等 較高
速度限制 可能略有延遲 正常 優先處理
適用場景 輕度使用、測試 日常個人助理 重度使用、高頻呼叫

💡 說明： TokenPlan 的優勢在於固定費用、不按量計費——用多少都在方案額度內，不用擔心帳單爆增。相較之下，Claude API 是按 token 用量計費（實際約 $10-20/天）。

主要模型：

模型 特色 速度 價格
MiniMax-M2.5 主力模型，中文理解優秀，支援 Agent 工具呼叫 ⭐⭐⭐⭐ 極低
MiniMax-M2-her 多角色沉浸扮演，適合角色扮演應用 ⭐⭐⭐⭐ 低

優點：

  • 固定費用：TokenPlan 不按量計費，用多少都在額度內，不怕帳單爆增
  • 中文理解優秀：對中文 Prompt 回應自然流暢
  • 固定費用：TokenPlan 不按量計費，用多少都在額度內
  • 整合多元能力：語音、影像、音樂模型一站式使用
  • 網路延遲低：在中國及台灣速度表現佳

缺點：

  • 英語任務有時略遜於 Claude
  • 需要中國手機號或 Email 註冊
  • Tool Use 工具呼叫能力比 Claude 稍新（還在快速迭代中）

3. 手把手安裝教學

💡 小技巧： 安裝過程中，隨時把遇到的錯誤訊息貼給任何 AI（ChatGPT、Claude 等）求解。把 AI 當成你的即時技術支援——它們非常擅長解讀錯誤訊息並給出修復步驟。

前置需求

  • Windows 11 電腦（需關閉 S Mode 才能裝 WSL2）
  • 網路連線
  • 一個 Claude API 帳號（見下方 Step 5）

流程總覽

步驟 內容 時間
1 安裝 WSL2（Linux 環境） 10 分鐘
2 修復 WSL2 DNS 問題 5 分鐘
3 安裝 Node.js 22 5 分鐘
4 安裝 OpenClaw 10 分鐘
5 選擇並申請 AI 模型 API（Claude 或 MiniMax） 5 分鐘
6 建立 Telegram 機器人 2 分鐘
7 設定精靈 + 配對 10 分鐘

Step 1：安裝 WSL2

WSL2 在 Windows 裡建立一個隔離的 Linux 環境，OpenClaw 在其中運作。你不需要學 Linux——大部時候只是複製貼上指令。

1a. 以系統管理員身份開啟 PowerShell

按 Win + X → 選「終端機（系統管理員）」

1b. 執行安裝：

wsl --install

安裝約 5-10 分鐘，完成後重新啟動電腦。重啟後 Ubuntu 終端機會開啟，要求建立 Linux 帳密（跟 Windows 帳號完全獨立）。

⚠️ 輸入密碼時畫面不會顯示任何字元，這是 Linux 正常行為，直接輸入按 Enter 即可。

Step 2：修復 WSL2 DNS 問題（重要！）

🚨 這是實際安裝中第一個坑。 WSL2 預設的 DNS 設定經常失效，導致後續所有下載失敗。強烈建議在裝任何東西之前先處理。

測試 DNS 是否正常：

ping -c 2 google.com

如果出現 name resolution 錯誤，依序執行以下修復：

sudo sh -c 'echo "nameserver 8.8.8.8" > /etc/resolv.conf'
sudo sh -c 'echo -e "[network]\ngenerateResolvConf = false" > /etc/wsl.conf'

然後在 Windows PowerShell 執行：

wsl --shutdown

重新開啟 Ubuntu 終端機，再測試一次 ping -c 2 google.com，成功即可繼續。

Step 3：安裝 Node.js 22

OpenClaw 需要 Node.js 22+，Ubuntu 預設版本太舊：

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

驗證：node --version 應顯示 v22.x.x 或更高。

Step 4：安裝 OpenClaw

⚠️ 專案歷經多次改名（Clawdbot → Moltbot → OpenClaw）。正確的安裝方式是 npm，不是 getclaw.dev 等舊腳本。

npm install -g openclaw

安裝完成後執行：

openclaw onboard --install-daemon

選擇 QuickStart，技能先 Skip（之後再安裝）。

Step 5：選擇並申請 AI 模型 API（選擇其一）

選項 A：Claude API（推薦有信用卡、複雜任務者）

  1. 前往 console.anthropic.com 註冊（跟 claude.ai 是不同的帳號系統）
  2. 左側選單 → Billing → 加入信用卡，設定月消費上限
  3. 左側選單 → API Keys → Create Key → 命名為 OpenClaw → 立即複製保存

⚠️ 金鑰只顯示一次，遺失只能重新建立。務必持續監控使用量，實際費用約 USD $10-20/天。

選項 B：MiniMax API（推薦台灣/中國用戶、追求性價比者）

  1. 前往 platform.minimax.io/subscribe/token-plan 選購 TokenPlan（不同方案有不同用量上限）
  2. 訂閱後進入 Dashboard → 左側「API Keys」→ 建立並複製 API Key
  3. 模型名稱使用 MiniMax-M2.5（或 MiniMax-M2.7 最新旗艦版）

💡 建議： 如果你在台灣、預算有限、主要做中文對話，從 MiniMax 開始是最佳選擇。如果需要最強推理能力、以英語為主，選 Claude API。

Step 6：建立 Telegram 機器人--未來就能用手機連絡你的龍蝦

  1. 在 Telegram(手機App即可) 搜尋 @BotFather → /start → /newbot
  2. 設定名稱和 username（username 必須以 bot 結尾）
  3. 儲存回覆的 Bot Token（千萬不要公開！）
  4. 同時搜尋 @userinfobot → /start → 記下你的數字 User ID

🚨 Bot Token 等於機器人的密碼，絕對不要公開。如果不慎洩漏，立刻去 BotFather 重新產生。

Step 7：執行設定精靈

在 Ubuntu 終端機執行以下指令啟動設定精靈：

openclaw onboard --install-daemon

選擇 QuickStart，技能先 Skip（之後再安裝）。

7a：設定 AI 模型

設定精靈會問你選擇哪個 AI 模型，依你的 API 選擇：

使用 Claude API： 1. 選擇 Anthropic / Claude → 貼上 Claude API Key 2. 選擇模型：claude-sonnet-4-5（快且省，推薦日常使用）或 claude-opus-4-5（最強，適合複雜任務）

使用 MiniMax API： 1. 選擇 MiniMax → 貼上 MiniMax API Key 2. 選擇模型：MiniMax-M2.5 或 MiniMax-M2.7（最新旗艦）

💡 Endpoint 不需要手動填寫——選擇 MiniMax 作為 Provider 後，Endpoint 會自動帶入正確的 https://api.minimax.io/anthropic。

7b：設定 Telegram 頻道

繼續設定精靈中 Telegram 頻道：

  1. 選擇 Telegram (Bot API)
  2. 貼上你的 Bot Token
  3. 確認你的數字 User ID（之前在 @userinfobot 取得的）

完成後確認頻道狀態：

openclaw channels status

應顯示 enabled, configured, running, mode:polling。

7c：配對你的 Telegram 帳號

最後一步：把你的 Telegram 帳號和 OpenClaw 配對。

  1. 在 Telegram 上找到你的 bot → 按 Start → 傳送任意訊息
  2. Bot 會回覆一組配對碼（例如 84KXQ9XM）
  3. 回到 Ubuntu 終端機執行：
openclaw pairing approve telegram <配對碼>

⚠️ 配對碼有一小時有效期限，逾期需重新發送訊息取得新碼。

配對成功後，bot 就會開始回覆你的訊息！試著傳一句「你好」測試看看。

安裝技能（Skills，可跳過但建議安裝）

直接在 Telegram 跟 bot 說「幫我安裝 Gmail skill」，它會引導你完成。

常用技能：

openclaw skill install @openclaw/gmail
openclaw skill install @openclaw/google-calendar
openclaw skill install @openclaw/files

Google 服務安裝時會開瀏覽器做一次 OAuth 登入，點「允許」即可。每次裝完新技能建議執行：openclaw gateway restart

Telegram 連線疑難排解

問題 原因與解法
setMyCommands 網路請求失敗 WSL2 DNS 沒修好，回到 Step 2 修復後 openclaw gateway restart
Token 有效但 bot 不回應 可能 webhook 衝突。執行 curl .../deleteWebhook 清除後重試
pairing required 重複循環 舊版 process 殘留。ps aux | grep openclaw 檢查後 restart
allowFrom 不生效 只接受數字 User ID，不接受 @username。執行 openclaw doctor --fix
401 Unauthorized Token 複製有隱藏字元。去 BotFather 重新產生，逐字確認

4. 打開與使用 Web 控制台

如何打開 Control UI

方法一：CLI 指令（最簡單）

openclaw dashboard

這會自動打開瀏覽器，並複製正確的 URL。

方法二：直接輸入

http://127.0.0.1:18789/

方法三：SSH Tunnel（遠端訪問）

ssh -N -L 18789:127.0.0.1:18789 user@vps-ip
# 瀏覽器打開 http://127.0.0.1:18789/

方法四：Tailscale（推薦，自動 HTTPS）

openclaw gateway --tailscale serve
# 然後用 Tailscale MagicDNS 訪問

第一次連線：Device Pairing

Step 1：複製 Token

在 Gateway 主機的終端執行：

openclaw config get gateway.auth.token

Step 2：打開 Control UI

瀏覽器打開 http://127.0.0.1:18789/，如圖所示填入：

  • WebSocket URL： ws://127.0.0.1:18789（已預填）
  • 網關令牌： 貼上剛才複製的 Token

⚠️ 如果看到紅色錯誤訊息「unauthorized: gateway token missing」，表示 Token 沒填或填錯了。

Step 3：批准設備（如果要求）

如果是第一次從某瀏覽器連線，會看到錯誤：

disconnected (1008): pairing required

在 Gateway 主機上批准：

openclaw devices list
openclaw devices approve <requestId>

5. OpenClaw 的自我調整：讓它學會你的習慣

5.1 Memory 系統：讓它記住你是誰

OpenClaw 沒有內建永久記憶——但它有 Memory 檔案，用來儲存你告訴它的資訊。

在 Workspace 的 MEMORY.md 寫入你的背景：

# MEMORY.md — 長期記憶

## 我的背景
- 姓名：Kevin
- 機器學習背景
- 偏好：希望 AI 直接執行，不要一直問問題

## 我的專案
- 目前的專案：半導體 wafer defect 分析平台
- 技術棧：Python + FastAPI + Vue

## 偏好設定
- 回覆語言：中文優先
- 複雜任務：先列出完整計畫再執行

5.2 Skills 系統：為它添加專業能力

Skills 是 OpenClaw 的技能擴展模組，讓它學會特定領域的專業知識。

官方技能：

Skill 功能
github 管理 GitHub Issues、PR、CI
weather 查天氣預報
coding-agent 委派複雜程式任務給 Codex
healthcheck 主機安全審計
tmux 遙控 tmux 終端

安裝方式： 在 Control UI → Skills 面板點擊安裝，或在 Telegram 跟 bot 說「安裝 XXX skill」。

5.3 Cron 排程：讓它主動出擊

Cron 任務讓 OpenClaw 在特定時間自動執行工作，不需要你每次提醒。

任務 頻率 說明
每日天氣預報 每天 09:00 上班前自動通知
每週一專案進度摘要 每週一 10:00 自動生成週報
每天檢查 GitHub 新 Issues 每 4 小時 有新問題主動通知

在 Control UI → Cron 面板設定，或用 CLI：

openclaw cron add \
  --name "每日天氣" \
  --schedule "0 9 * * *" \
  --session-target isolated \
  --message "告訴 Kevin 今天台北的天氣" \
  --delivery announce

5.4 Heartbeat：讓它定期檢查

在 Workspace 建立 HEARTBEAT.md，設定定期檢查清單：

  • 檢查郵件（有無重要未讀？）
  • 檢查行事曆（2小時內有約會嗎？）
  • 天氣（今天適合出門嗎？）

告訴它什麼時候可以打擾你（白天）、什麼時候保持安靜（晚上 23:00 - 早上 08:00）。

安全性注意事項

🔐 OpenClaw 對你的電腦有檔案讀寫和指令執行的權限，請謹慎使用。

  • 使用專用電腦 — 不要用存有個人資料的主力機器（尤其是信用卡等付款資訊）
  • 建立獨立帳號 — 建議建立一個獨立的 Windows 使用者帳號讓 OpenClaw 使用
  • 絕對不要公開 Bot Token — 等於機器人的密碼
  • 為 Telegram 開啟兩步驟驗證
  • 在 API 提供商設定月消費上限
  • 定期備份 ~/.openclaw（所有憑證和記憶都在裡面）
  • 儀表板（localhost:18789）不要對外公開
